# Glossário

Termos técnicos e conceitos usados neste guia.

---

## A

**ADR (Architecture Decision Record)**: Documento que registra uma decisão arquitetural importante, incluindo contexto, alternativas e trade-offs.

**ARIA**: Accessible Rich Internet Applications. Especificação W3C para tornar conteúdo web acessível.

## C

**C4 Model**: Modelo de diagramação de arquitetura em 4 níveis: Contexto, Containers, Componentes, Código.

## D

**Definition of Done (DoD)**: Checklist de critérios que uma feature deve atender para ser considerada "pronta".

## G

**Gherkin**: Linguagem para escrever critérios de aceitação em formato legível (Given/When/Then).

## J

**JTBD (Jobs to be Done)**: Framework que foca no "trabalho" que o usuário quer realizar, não na solução.
- Formato: "Quando [situação], quero [ação] para [resultado]"

## M

**MoSCoW**: Técnica de priorização:
- **Must have**: Obrigatório
- **Should have**: Importante
- **Could have**: Desejável
- **Won't have**: Fora do escopo (por enquanto)

**MVP (Minimum Viable Product)**: Versão mínima do produto com funcionalidades essenciais para validar hipóteses.

## N

**North Star Metric**: Métrica principal que indica sucesso do produto e alinhamento com valor entregue ao usuário.

## O

**OKR (Objectives and Key Results)**: Framework de definição de metas com objetivos qualitativos e resultados-chave mensuráveis.

**OWASP**: Open Web Application Security Project. Organização que publica guias de segurança, incluindo o OWASP Top 10.

## P

**PRD (Product Requirements Document)**: Documento de requisitos do produto, incluindo visão, personas, funcionalidades e escopo.

## R

**RICE**: Framework de priorização:
- **R**each: Quantas pessoas impacta
- **I**mpact: Qual o impacto por pessoa
- **C**onfidence: Quão confiante estamos
- **E**ffort: Quanto esforço requer

## V

**Vibe Coding**: Termo para desenvolvimento rápido com IA, no "fluxo". Este guia propõe o **Vibe Coding Estruturado**, que adiciona engenharia de software ao processo.

## W

**WCAG**: Web Content Accessibility Guidelines. Diretrizes de acessibilidade web do W3C. Níveis: A, AA, AAA.
