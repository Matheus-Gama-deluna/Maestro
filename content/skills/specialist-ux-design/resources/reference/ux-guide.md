# Guia Completo de UX Design - Maestro Skills

## 🎯 Visão Geral
Guia com princípios, frameworks e fluxos para o especialista de UX Design do Maestro. Baseado em Design Thinking, UXR moderno e melhores práticas de 2025.

---

## 1. Fundamentos de UX

### 1.1 Processo
```
Descobrir → Definir → Idear → Prototipar → Validar → Iterar
```

### 1.2 Princípios
- **User-Centered:** Decisões baseadas em pessoas reais
- **Consistência:** Sistema de design unificado
- **Acessibilidade:** WCAG 2.1 AA como padrão mínimo
- **Performance:** Design leve e responsivo
- **Escalabilidade:** Componentização e reuso

---

## 2. Pesquisa e Personas

### 2.1 Técnicas de Pesquisa
- Entrevistas moderadas/rápidas
- Shadowing remoto
- Surveys quantitativos
- Análise de analytics

### 2.2 Personas
| Campo | Descrição |
|-------|-----------|
| Nome | Humanize o perfil |
| Objetivos | O que tentam alcançar |
| Dores | Barreiras atuais |
| Motivadores | Por que usariam |
| Cenários | Contexto de uso |

---

## 3. Jornadas e Fluxos

### 3.1 Jornada Completa
1. Descoberta
2. Consideração
3. Conversão
4. Retenção
5. Advocacy

### 3.2 Mapeamento por Etapa
| Etapa | Emoção | Pontos de contato | Oportunidades |
|-------|--------|-------------------|---------------|

### 3.3 Prioridade
- Foque em jornadas críticas primeiro
- Mapeie dores e ganhos na mesma linha
- Conecte com métricas (North Star/OKRs)

---

## 4. Arquitetura de Informação

### 4.1 Estrutura
- Card sorting / tree testing
- Sitemap com hierarquia clara
- Navegação consistente em todos os dispositivos

### 4.2 Fluxos
- User flows por persona
- Task flows para ações críticas
- Edge cases documentados

---

## 5. Wireframes e Prototipagem

### 5.1 Wireframes
- Low → Mid fidelity para alinhamento rápido
- High fidelity somente após validação
- Componentização via Atomic Design (Atom → Molecule → Organism)

### 5.2 Prototipagem
- Figma como padrão (modo protótipo)
- Microinterações básicas (hover, focus, loading)
- Links atualizados no `design-doc.md`

---

## 6. Design Visual

### 6.1 Style Guide
- Paleta definida (Primária, Secundária, Feedback)
- Tipografia (Headings, Body, Caption)
- Iconografia e ilustrações padronizadas

### 6.2 Responsividade
| Breakpoint | Estratégia |
|------------|------------|
| Mobile | Stack vertical / bottom nav |
| Tablet | Grid em 2 colunas |
| Desktop | Grid completo + painéis |

---

## 7. Acessibilidade

### 7.1 WCAG 2.1 AA
- Contraste 4.5:1 para texto normal
- Foco visível em todos os elementos navegáveis
- Navegação keyboard-friendly
- Labels semânticos para leitores
- Conteúdo adaptável (sem zoom fixo)

### 7.2 Checklist Rápido
- [ ] Texto alternativo em imagens
- [ ] Ordem lógica de navegação
- [ ] Mensagens claras de erro
- [ ] Instruções não dependem apenas de cor
- [ ] Teste com ferramentas (axe, WAVE)

---

## 8. UX Writing

### 8.1 Microcopy
- Claro, direto, conversacional
- Ações descritivas ("Continuar" ≠ "Salvar")
- Mensagens de erro com solução acionável

### 8.2 Tom e Voz
| Contexto | Tom |
|----------|-----|
| Onboarding | Acolhedor |
| Emergências | Calmo e objetivo |
| Sucesso | Empolgante |

---

## 9. Métricas e Validação

### 9.1 KPIs
- Task Success Rate
- Time on Task
- CSAT / NPS por etapa
- Taxa de conversão por jornada
- SUS (System Usability Scale)

### 9.2 Testes
- Usabilidade moderada (5 participantes)
- Guerrilla tests para fluxos simples
- Testes A/B para otimização contínua

---

## 10. Integração com Squad

### 10.1 Artefatos Entregues
- Documento consolidado `design-doc.md` contendo wireframes e jornadas

- Score de validação ≥ 75
- Checklist completo
- Feedback de stakeholders incorporado
- Protótipos aprovados
- Contexto atualizado para Modelagem de Domínio
